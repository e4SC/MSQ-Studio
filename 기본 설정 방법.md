# ⚙️ 맵 기본 설정 사용 설명서
## 처음 사용하는 분도 그대로 따라할 수 있는 전체 안내서
### 맵의 모든 기본 설정은 프로그램의 맵 설정 에서 합니다.

이 문서는 **맵 설정 기능을 처음 사용하는 사용자 기준**으로 작성된 설명서입니다.  
프로그램 화면에서 실제로 보이는 항목을 기준으로, **무엇을 어디에 입력해야 하는지**, **어떤 옵션이 어떤 역할을 하는지**, **어떤 값은 꼭 입력해야 하는지**를 한 번에 이해할 수 있도록 정리했습니다.

---

# 1. 맵 설정에서 할 수 있는 것

맵 설정에서는 게임 전체에 영향을 주는 항목들을 관리할 수 있습니다.

대표적으로 아래 내용을 설정합니다.

- 스피드 모드 / 감상 모드의 힌트 출력 시간
- 주관식 힌트 / 2차 힌트 시간
- 오프닝 / 엔딩 시간
- 주제 / 노래 / 가수 / 주관식 / 객관식 사용 여부
- 셔플, 장르 선택, 정답 공개, 랭킹, 익명, 자동다음곡 같은 게임 옵션
- 기본 UI / 디자인 UI 선택
- 스킵 관련 키 지정
- 맵 이름 / 맵 설명 / 포스 이름
- 효과음 파일 지정
- 리더보드 문구
- 문항 이름, 색상, 점수
- 연결된 맵 / 저장 맵 경로 지정

즉, **게임 규칙 + 화면 표시 + 맵 메타정보 + 효과음 + 키 설정**을 한 번에 관리하는 메뉴라고 생각하시면 됩니다.

---

# 2. 먼저 알아두면 좋은 핵심

## 2-1. 이 메뉴는 “게임 전체 설정”입니다
노래 하나하나를 수정하는 곳이 아니라,  
**게임 전체가 어떻게 진행될지**를 정하는 곳입니다.

예:
- 힌트를 몇 초 뒤에 띄울지
- 어떤 정답 종류를 쓸지
- 오프닝이 몇 초 동안 나올지
- 정답 맞히면 몇 점을 줄지

---

## 2-2. 저장하지 않으면 반영되지 않습니다
화면에서 값을 바꾸기만 해서는 끝나지 않고,  
맨 아래의 **저장 버튼**을 눌러야 실제로 적용됩니다.

닫을 때 저장하지 않은 변경사항이 있으면 저장 여부를 다시 물어봅니다.

---

## 2-3. 일부 옵션은 서로 같이 쓸 수 없습니다
대표적으로 다음 조합은 주의가 필요합니다.

- **주관식 / 객관식**
- **기본 UI / 디자인 UI**
- **디자인 UI / 객관식**

즉, 어떤 옵션을 켜면 다른 옵션이 자동으로 꺼지거나 사용할 수 없게 될 수 있습니다.

---

# 3. 화면 구성

맵 설정 창은 크게 **왼쪽 영역**과 **오른쪽 영역**으로 나뉩니다.

- **왼쪽**: 게임 진행 / 기능 / 옵션 / 파일 설정 / 효과음
- **오른쪽**: 힌트 / 오프닝 / 엔딩 / 리더보드 / 키 설정 / 문항 이름 / 맵 경로

---

# 5. 왼쪽 영역 설명

---

## 5-1. 스피드 모드 힌트 출력 시간

이 영역은 **스피드 모드에서 힌트가 언제 나올지** 정하는 부분입니다.

총 4칸이 있습니다.

각 칸에는 숫자를 입력합니다.

이 값들은 스피드 모드에서 **힌트가 몇 초 후에 공개되는지**를 결정합니다.

### 주의
- 숫자만 입력해야 합니다
- 비워두면 안 됩니다
- 잘못 입력하면 저장할 때 오류가 납니다

---

## 5-2. 감상 모드 힌트 출력 시간

이 영역은 **감상 모드에서 힌트가 언제 공개될지** 정하는 부분입니다.

항목은 다음과 같습니다.

- 주제 힌트
- 노래 힌트
- 가수 힌트
- 주/객관식 문제 출력

각 항목별로 몇 초 뒤에 나올지 숫자로 입력합니다.

### 예시 이해
- 주제 힌트 5초
- 노래 힌트 10초
- 가수 힌트 15초
- 문제 출력 20초

처럼 설정하면, 감상 모드에서 시간이 흐르면서 순서대로 공개됩니다.

---

## 5-3. 주관식 힌트 / 2차 힌트

왼쪽 중간쯤에는 추가 힌트 시간을 넣는 칸이 있습니다.

- **주관식 힌트**
- **2차 힌트**

### 주관식 힌트
주관식 문제를 사용할 때, 몇 초 후에 주관식 힌트를 띄울지 정합니다.

### 2차 힌트
2차 힌트를 사용할 경우, 몇 초 후에 추가 힌트를 띄울지 정합니다.

### 중요한 규칙
프로그램은 저장할 때 시간 관계를 검사합니다.

예를 들어:
- 주관식 힌트는 문제 출력 시간보다 늦어야 함
- 2차 힌트는 주관식 힌트보다 늦어야 함

즉, 앞 단계보다 뒤에 나오는 힌트로 설정해야 합니다.

---

## 5-4. 기능 설정

여기는 게임에서 어떤 종류의 정보를 사용할지 정하는 핵심 영역입니다.

체크할 수 있는 항목은 다음과 같습니다.

- 음악 셔플
- 장르 선택 사용
- 2차 힌트 사용
- 주제 사용
- 노래 사용
- 가수 사용
- 주관식 사용
- 객관식 사용
- 음악 재생 딜레이

---

### 음악 셔플
곡 순서를 랜덤하게 섞습니다.

### 장르 선택 사용
게임 시작 시 장르를 고를 수 있게 합니다.

### 2차 힌트 사용
2차 힌트를 사용할지 정합니다.  
켜면 2차 힌트 입력칸이 실제로 사용됩니다.

### 주제 / 노래 / 가수 사용
정답 종류로 주제, 노래, 가수를 사용할지 정합니다.

### 주관식 사용
주관식 문제를 사용할지 정합니다.

### 객관식 사용
객관식 문제를 사용할지 정합니다.

### 음악 재생 딜레이
곡과 곡 사이가 바로 이어지지 않도록 할지 정합니다.

---

## 5-5. 주관식 / 객관식 사용 규칙

이 부분은 특히 중요합니다.

### 기본 UI일 때
- 주관식과 객관식은 서로 같이 켜둘 수 없는 구조로 동작할 수 있습니다
- 한쪽을 켜면 다른 쪽이 자동으로 꺼질 수 있습니다

### 디자인 UI일 때
- 객관식은 사용할 수 없게 됩니다
- 즉, 디자인 UI를 쓰면 주관식 중심으로 진행된다고 생각하시면 됩니다

### 최소 1개 이상 필요
주제 / 노래 / 가수 / 주관식 / 객관식 중  
**최소 하나 이상은 사용해야 저장할 수 있습니다.**

---

## 5-6. 게임 옵션

이 영역에서는 진행 방식과 표시 방식을 정합니다.

항목은 다음과 같습니다.

- 랭킹
- 익명
- 정답공개
- 자동다음곡

---

### 랭킹
플레이어 이름 앞에 등수를 표시합니다.

예:
- 1등
- 2등

### 익명
플레이어 닉네임 대신 익명 형태로 표시합니다.

예:
- 플레이어1
- 플레이어2

### 정답공개
문제가 끝났을 때 남은 정답을 자동으로 공개합니다.

### 자동다음곡
문제가 끝난 뒤 자동으로 다음 곡으로 넘어가는 기능입니다.

체크하면 남은 시간이 자동으로 고정됩니다.  
체크하지 않으면 직접 시간을 입력할 수 있습니다.

---

## 5-7. 자동다음곡 시간

자동다음곡을 켜지 않았을 때는  
옆 입력칸에 시간을 직접 넣을 수 있습니다.

### 입력 규칙
- 공백 가능
- 숫자 가능
- 허용 범위: **4 ~ 999**
- 공백이면 별도 기본 동작으로 처리됩니다

### 자동다음곡을 켜면
- 시간이 자동으로 설정되고
- 직접 입력칸은 비활성화됩니다

---

## 5-8. UI 설정

여기서는 게임 화면 스타일을 정합니다.

- **기본 UI**
- **디자인 UI**

둘 중 하나만 사용할 수 있습니다.

### 기본 UI
전통적인 기본 UI 형태입니다.

### 디자인 UI
좀 더 꾸며진 형태의 UI입니다.

### 중요한 제한
- 디자인 UI를 선택하면 객관식 사용이 비활성화됩니다
- 즉, 객관식을 쓸 예정이면 기본 UI를 선택해야 합니다

---

## 5-9. 스킵 설정

스킵 관련 옵션을 정하는 영역입니다.

현재 보이는 대표 옵션은:

- **과반수 스킵**

### 과반수 스킵
여러 명이 플레이할 때, 과반수가 동의해야 스킵이 되도록 설정합니다.

체크하지 않으면 보다 자유로운 스킵 방식으로 진행됩니다.

---

## 5-10. 파일 설정

게임방 목록 등에서 보이는 맵 정보를 정하는 곳입니다.

입력 항목:
- 파일 이름
- 파일 설명
- 포스1
- 포스2

---

### 파일 이름
방 만들 때 보이는 맵 이름입니다.

### 파일 설명
맵 설명에 표시되는 문구입니다.

### 포스1 / 포스2
포스 이름을 설정합니다.

### 주의
이 4개는 비워둘 수 없습니다.  
하나라도 비어 있으면 저장이 되지 않습니다.

---

## 5-11. 효과음 설정

왼쪽 아래에는 효과음을 지정하는 버튼들이 있습니다.

항목:
- 오프닝1
- 오프닝2
- 클릭
- 스킵
- 정답

각 버튼을 눌러 효과음 파일을 선택할 수 있습니다.

### 역할 설명
- **오프닝1**: 첫 번째 오프닝 멘트와 함께 재생
- **오프닝2**: 두 번째 오프닝 멘트와 함께 재생
- **클릭**: 클릭, 객관식 투표, 스킵 투표 등에 사용
- **스킵**: 스킵 시 재생
- **정답**: 정답을 맞췄을 때 재생

### 파일 선택 규칙
- 지정된 효과음 폴더 안의 파일만 사용
- 지원 형식: **wav / ogg**
- 모든 효과음을 반드시 지정해야 저장할 수 있습니다

---

# 6. 오른쪽 영역 설명

---

## 6-1. 주관식 힌트 / 2차 힌트

오른쪽 상단에도 힌트 시간 관련 항목이 있습니다.

- 주관식 힌트
- 2차 힌트

이 값은 실제 저장 시 중요한 검사 대상입니다.

### 예시 규칙
- 주관식 문제를 쓸 경우, 주관식 힌트 시간은 문제 출력 시간보다 뒤여야 함
- 2차 힌트를 쓸 경우, 2차 힌트는 주관식 힌트보다 뒤여야 함

즉, 힌트 순서를 거꾸로 넣으면 저장이 막힙니다.

---

## 6-2. 오프닝 / 엔딩 시간

항목:
- 오프닝1 시간
- 오프닝2 시간
- 오프닝3 시간
- 엔딩 시간

각 항목에는 **초 단위 숫자**를 입력합니다.

### 의미
- 오프닝1: 첫 번째 오프닝 문구가 표시되는 시간
- 오프닝2: 두 번째 오프닝 문구가 표시되는 시간
- 오프닝3: 마지막 오프닝 문구가 표시되는 시간
- 엔딩: 엔딩 문구가 표시되는 시간

### 주의
모두 필수 입력입니다.  
비워두거나 숫자가 아니면 저장되지 않습니다.

---

## 6-3. 리더보드

오른쪽 중간에는 **리더보드 문구**를 넣는 칸이 있습니다.

이곳에 입력한 문구가 게임 중 리더보드 표시용 텍스트로 사용됩니다.

예:
- 최고 점수
- 현재 순위
- 라운드 점수

### 주의
비워두면 저장되지 않습니다.

---

## 6-4. 키 지정

여기서는 게임 중 사용할 단축키를 정합니다.

항목:
- 스킵 투표
- 자동 스킵
- 방장 스킵
- 방장 객관식 스킵
- 방장 자동 스킵 해제

### 실제 사용 시 주의
현재 구조상 일부 자동 스킵 관련 항목은 사실상 비활성 상태로 안내될 수 있습니다.  
즉, 화면에 있어도 실제 사용이 제한되는 항목이 있을 수 있습니다.

### 반드시 기억할 점
키는 서로 중복되면 안 됩니다.

예:
- 스킵 투표 = A
- 방장 스킵 = A

이렇게 같은 키를 쓰면 저장할 때 오류가 납니다.

---

## 6-5. 키 선택 방법

키 선택 칸은 목록에서 골라 사용할 수 있습니다.

입력 가능한 예:
- 알파벳
- 숫자
- 방향키
- 기능키(F1~)
- 일부 특수키

### 주의
필수 키가 비어 있으면 저장할 수 없습니다.

---

## 6-6. 문항 이름 / 색상 / 점수

이 영역은 실제 게임에서 표시되는 문구와 점수를 정하는 곳입니다.

예를 들어:
- 노래
- 가수
- 문제
- 객관식
- 주관식 정답 수

같은 텍스트를 사용자가 보기 좋게 바꿀 수 있습니다.

각 줄마다 설정 가능한 것은 다음과 같습니다.

- 문항 이름
- 색상
- 점수
- 미리보기

---

### 문항 이름
화면에 표시될 텍스트를 정합니다.

예:
- 노래 → 곡명
- 가수 → 아티스트
- 문제 → 퀴즈

일부 항목은 수정이 제한될 수 있습니다.

---

### 색상
각 문항 텍스트의 색을 정합니다.

예:
- 노래는 노랑
- 가수는 주황
- 문제는 초록

선택한 색은 옆 미리보기에도 반영됩니다.

---

### 점수
각 문항을 맞췄을 때 줄 점수입니다.

예:
- 주제 1점
- 노래 1점
- 가수 2점
- 문제 1점
- 객관식 2점

### 주의
- 점수는 숫자여야 합니다
- 0보다 커야 합니다
- 비워두면 저장되지 않습니다

---

## 6-7. 연결된 맵 / 저장 맵

오른쪽 아래에는 맵 파일을 지정하는 항목이 있습니다.

- **연결된맵**
- **저장맵**

각 항목 옆의 **찾기 버튼**으로 `.scx` 파일을 선택할 수 있습니다.

### 연결된맵
현재 작업과 연결할 맵 파일입니다.

### 저장맵
실제로 저장 대상이 되는 맵 파일입니다.

### 파일 선택 규칙
- `.scx` 파일만 선택 가능
- 프로그램 폴더 안에 있으면 파일명만 저장
- 다른 경로면 전체 경로로 저장될 수 있음

---

# 7. 저장 버튼

오른쪽 맨 아래에 있는 **저장** 버튼을 누르면  
현재 설정값을 실제 파일들에 반영합니다.

저장 시에는 다음이 함께 처리됩니다.

- 게임 설정 저장
- UI / 점수 / 색상 정보 저장
- 효과음 설정 저장
- 키 설정 저장
- 맵 이름 / 설명 / 포스 저장
- 연결된 맵 / 저장 맵 경로 저장

저장이 끝나면 완료 메시지가 뜹니다.

---

# 8. 저장할 때 검사하는 것

맵 설정은 저장할 때 꽤 많은 검사를 합니다.

---

## 8-1. 비어 있으면 안 되는 항목

대표적으로 아래는 비어 있으면 안 됩니다.

- 파일 이름
- 파일 설명
- 포스1
- 포스2
- 리더보드
- 오프닝1~3 / 엔딩 시간
- 필수 키
- 효과음
- 점수
- 문항 이름
- 색상 선택

---

## 8-2. 숫자만 들어가야 하는 항목

다음은 숫자여야 합니다.

- 스피드 모드 힌트 시간
- 감상 모드 힌트 시간
- 주관식 힌트 시간
- 2차 힌트 시간
- 오프닝 / 엔딩 시간
- 자동다음곡 시간
- 점수

숫자가 아니면 저장이 막힙니다.

---

## 8-3. 힌트 시간 순서 검사

주관식 문제를 사용할 경우:

- 문제 출력 시간보다 주관식 힌트가 늦어야 함
- 사용 중인 주제 / 노래 / 가수 힌트는 문제 출력 시간보다 빨라야 함
- 2차 힌트는 주관식 힌트보다 늦어야 함

주관식을 사용하지 않을 경우:

- 2차 힌트가 주제 / 노래 / 가수 힌트보다 늦어야 함

즉, 힌트 공개 순서를 말이 되게 설정해야 합니다.

---

## 8-4. 문제 종류 최소 1개 검사

다음 중 최소 하나는 반드시 켜져 있어야 합니다.

- 주제
- 노래
- 가수
- 주관식
- 객관식

전부 끄면 저장할 수 없습니다.

---

## 8-5. 키 중복 검사

서로 다른 기능에 같은 키를 주면 저장되지 않습니다.

예:
- 스킵 투표 = F1
- 방장 스킵 = F1

이렇게 중복되면 오류가 납니다.

---

# 9. UI와 문제 종류 조합 설명

이 부분은 가장 많이 헷갈립니다.

---

## 9-1. 기본 UI
- 객관식 사용 가능
- 주관식 / 객관식 조합 관리 가능

## 9-2. 디자인 UI
- 객관식 사용 불가
- 주관식 중심

즉:

> **객관식을 쓰려면 기본 UI를 선택해야 합니다.**

---

# 10. 장르 선택이 비활성화되는 경우

장르 선택은 특정 조합에서 사용할 수 없게 될 수 있습니다.

대표적으로:
- 기본 UI
- 객관식만 사용
- 주제 / 노래 / 가수 / 주관식 전부 사용 안 함

같은 조합에서는 장르 선택이 자동으로 꺼질 수 있습니다.

즉, 장르로 고를 대상이 없는 상황에서는 장르 선택도 의미가 없어지기 때문입니다.

---

# 11. 저장하지 않고 닫을 때

값을 바꾼 뒤 저장하지 않고 창을 닫으려고 하면  
프로그램이 다음 중 하나를 묻습니다.

- 저장
- 저장 안 함
- 취소

### 저장
현재 값을 저장하고 닫습니다.

### 저장 안 함
저장하지 않고 닫습니다.

### 취소
닫기를 취소하고 계속 편집합니다.

# 13. 효과음 선택 시 주의사항

효과음은 아무 파일이나 되는 것이 아니라,  
지정된 위치의 wav / ogg 파일만 선택 가능합니다.

즉:
- mp3는 효과음용 선택 대상이 아님
- 폴더 밖 파일은 거부될 수 있음

모든 효과음이 채워져야 저장 가능합니다.

---

# 14. 맵 파일 지정 시 주의사항

연결된 맵 / 저장 맵은 `.scx` 파일만 선택 가능합니다.

보통은:
- 연결할 원본 맵
- 최종 저장 대상 맵

을 각각 지정하는 용도로 생각하시면 됩니다.

# 15. 툴팁으로 도움받기

이 화면의 대부분 항목에는 마우스를 올렸을 때 설명이 뜨도록 되어 있습니다.

즉:
- 주제 사용
- 노래 사용
- 가수 사용
- 2차 힌트 사용
- 점수
- 효과음
- 키 지정

등은 마우스를 올려서 짧은 도움말을 확인할 수 있습니다.

처음 사용할 때는 **툴팁을 함께 보면서 설정하는 것**을 추천합니다.

# 16. 단축키

맵 설정 창에서는 기본적으로 다음 동작이 가능합니다.

- **Enter** → 저장
- **Esc** → 닫기

즉, 저장은 빠르게 Enter로 할 수 있고,  
나가기만 할 때는 Esc를 사용할 수 있습니다.

---

이 순서대로 하면 대부분의 설정을 문제없이 끝낼 수 있습니다.
