# 💬 챗 이벤트 관리 창 사용법 (초보자용)

이 창은 **인게임 채팅 정답(별칭/동의어)을 표로 정리하고**, 최종적으로 **챗 이벤트 파일을 자동 생성**하는 도구입니다.  
처음 사용하는 분도 그대로 따라 할 수 있도록 **사용자 입장에서 처음부터 끝까지** 정리했습니다.

---

## 1) 이 창에서 할 수 있는 것

이 창에서는 다음 작업을 할 수 있습니다.

- 채팅으로 인정할 **정답 텍스트**를 표 형태로 입력
- 같은 의미의 답안을 **한 줄에 묶어서 관리**
- 같은 답안을 **다른 문제에서도 중복 사용**
- 중복으로 사용 중인 답안을 **표에서 바로 확인**
- 검색으로 원하는 답안 빠르게 찾기
- 여러 셀 복사 / 붙여넣기 / 삭제
- 작업 내용을 **백업 파일로 저장**
- 예전 백업 파일을 열어 현재 작업과 비교
- 백업창에서 현재 편집 중인 표로 **추가정답 자동 복사**
- 열(추가정답 칸) 늘리기 / 줄이기
- 작업 중 입력 내용을 미리 확인
- 최종적으로 챗 이벤트 파일 생성

---

## 2) 최근 업데이트로 추가된 기능

### 중복 정답 인식 기능 추가
1.5.0 버전부터는 **다른 문제에 사용한 정답도 중복으로 사용할 수 있습니다.**

즉, 예전처럼  
“이 답안은 다른 문제에서 이미 썼으니 입력 불가”가 아니라,

이제는 그냥 챗 이벤트 창에서 입력하면 되고,  
프로그램이 알아서 **중복 답안으로 인식해서 자동 적용**합니다.

중복으로 사용 중인 답안은 표에서 **`!` 표시가 붙습니다.**  
이 표시는 **경고가 아니라 확인 표시**입니다.  
즉, “이 답안은 다른 곳에서도 사용 중이다”라는 뜻입니다.

---

### 백업창 → 메인창 추가정답 자동복사 기능 추가
백업창에 **`일치 행 추가정답 자동복사`** 버튼이 추가되었습니다.

이 기능은:

- 백업 파일의 **기본정답**
- 현재 메인창의 **기본정답**

이 정확히 같은 행을 자동으로 찾아서,  
**추가정답만 한 번에 복사**해 줍니다.

추가로:

- 백업 쪽 열 수가 더 많으면 메인창의 열도 자동으로 늘어납니다.
- 백업에 없는 기본정답은 자동으로 건너뜁니다.
- 건너뛴 항목이 있으면, 복사 후 그 항목 위치로 이동해 확인하기 쉽게 해 줍니다.

---

### 텍스트 미리보기에서 현재 글자수 표시
셀을 편집할 때 위쪽에 뜨는 미리보기에서  
**현재 입력한 글자 수를 같이 확인**할 수 있게 되었습니다.

예:
- `아이유 (3자)`
- `tokinosora (10자)`

---

### 열너비 자동맞춤 기능 추가
상단의 **열너비 자동맞춤** 옵션을 켜면,  
각 열 안에서 **가장 긴 정답 길이**를 기준으로 열 너비가 자동 조절됩니다.

특징:

- 보기 편하게 열 폭이 자동으로 맞춰집니다.
- 체크를 해제하면 원래 사용하던 수동 너비로 돌아갑니다.
- 자동맞춤 체크 상태는 프로그램을 껐다 켜도 유지됩니다.
- Undo / Redo 이후에도 열너비가 이상하게 줄어들거나 초기화되지 않도록 개선되었습니다.

## 6) 기본 개념: 한 줄 = 같은 정답 그룹

**한 행(가로 한 줄)** 은  
서로 같은 정답으로 인정할 텍스트를 묶는 곳입니다.

예를 들어 한 줄에 이렇게 넣으면:

- `아이유`
- `IU`
- `이지은`

이 세 가지는 모두 같은 정답으로 인정됩니다.

즉,  
**같은 뜻 / 같은 사람 / 같은 곡 / 같은 표현이라면 한 줄에 넣으면 됩니다.**

---

## 7) 열의 의미

첫 번째 칸은 보통 **기본정답**입니다.  
그 뒤 칸들은 **추가정답**입니다.

예:
- 기본정답: `토키노소라`
- 추가정답1: `소라`
- 추가정답2: `tokinosora`

이렇게 입력하면 세 표현이 모두 같은 정답으로 처리됩니다.

---

## 8) 추가정답 열 늘리기 / 줄이기

### 추가정답 열 +
별칭이 더 많이 필요할 때 누릅니다.  
추가정답 칸이 하나 더 늘어납니다.

---

### 추가정답 열 -
마지막 추가정답 열을 없앱니다.

주의:
- 기본으로 있는 칸은 삭제할 수 없습니다.
- 마지막 열에 내용이 들어 있으면 정말 지울지 한 번 더 물어봅니다.

---

## 9) 입력 규칙

### 글자 수 제한
한 칸에 입력할 수 있는 길이는 **최대 26자**입니다.

26자를 넘으면 입력할 수 없습니다.

이 제한은:
- 입력 중에도 검사되고
- 최종 생성 전에도 다시 검사됩니다.

---

### 빈칸 허용
모든 칸을 꼭 채울 필요는 없습니다.  
필요한 만큼만 입력하면 됩니다.

---

### 같은 줄 안의 정답 정리
중간 칸을 지우면 뒤쪽 값이 앞으로 당겨져서  
추가정답 칸이 깔끔하게 정리됩니다.

예:
- 추가정답1 = A
- 추가정답2 = B
- 추가정답3 = C

여기서 B를 지우면 자동으로:

- 추가정답1 = A
- 추가정답2 = C

처럼 정리됩니다.

---

## 10) 중복 정답 기능

이 부분은 꼭 알아두시면 좋습니다.

### 예전과 달라진 점
이제는 다른 문제에 사용한 답도 **중복 사용 가능**합니다.

즉:
- 같은 답안을 여러 문제에서 쓸 수 있습니다.
- 따로 복잡하게 처리할 필요 없이 그냥 입력하면 됩니다.

---

### 중복 답안 표시
중복으로 사용 중인 답안은 표에서 **`!` 표시**가 붙습니다.

이건 오류 표시가 아니라  
**“이 답안은 다른 데에서도 쓰고 있음”을 보여주는 확인 표시**입니다.

즉, 잘못된 게 아니라  
프로그램이 정상적으로 중복 인식을 하고 있다는 뜻입니다.

---

### 사용자 입장에서 기억할 점
- 중복 답안을 써도 됩니다.
- 표에 `!` 표시가 보이면 “중복으로 쓰이는 답안이구나” 정도로 보면 됩니다.
- 프로그램이 알아서 처리하므로 별도 수작업은 필요 없습니다.

## 11) 셀 편집 시 미리보기

길이를 초과하기 전에 미리 확인할 수 있어서 편합니다.

## 13) 복사 / 붙여넣기 / 삭제

### 복사
- `Ctrl + C`

선택한 셀을 복사합니다.

---

### 붙여넣기
- `Ctrl + V`

붙여넣기는 두 가지 방식으로 동작합니다.

#### 1) 여러 칸에 같은 값 붙여넣기
여러 셀을 선택한 상태에서 한 줄짜리 텍스트를 붙여넣으면  
선택한 모든 칸에 같은 값이 들어갑니다.

#### 2) 표 형태로 붙여넣기
엑셀처럼 탭/줄바꿈이 들어 있는 데이터를 붙여넣으면  
현재 칸을 시작점으로 행과 열에 맞춰 들어갑니다.

---

### 잘라내기
- `Ctrl + X`

복사 후 선택된 셀 내용을 비웁니다.

---

### 삭제
- `Delete`

선택한 셀 내용을 비웁니다.

---

### 우클릭 메뉴
마우스 오른쪽 버튼으로도 다음 작업을 할 수 있습니다.

- 복사
- 붙여넣기
- 선택 셀 비우기

백업창에서는 수정이 불가능하므로 일부 기능은 제한됩니다.

---

## 14) Enter 키 이동 방식

이 창은 Enter 동작이 일반 문서 편집기와 조금 다릅니다.

현재 칸 편집을 마치고 Enter를 누르면  
다음 행으로 이동한 뒤, 그 행에서 **가장 왼쪽의 빈칸**으로 이동합니다.

즉, 빠르게 아래로 내려가며 입력하기 편하게 되어 있습니다.

---

## 15) Undo / Redo

실수로 수정했을 때 되돌릴 수 있습니다.

- `Ctrl + Z` : 실행 취소
- `Ctrl + Y` : 다시 실행

이 기능은 단순히 글자만 되돌리는 게 아니라,  
표 상태와 열 너비 같은 편집 상태도 함께 복구됩니다.

---

## 16) 열너비 자동맞춤

상단의 **열너비 자동맞춤** 체크를 켜면,  
표 안에서 가장 긴 답안 기준으로 열 폭이 자동 조절됩니다.

이 기능은 다음 상황에서 특히 편합니다.

- 긴 영어 별칭이 많을 때
- 짧은 정답/긴 정답이 섞여 있을 때
- 열 폭을 일일이 손으로 맞추기 귀찮을 때

특징:
- 자동으로 보기 좋은 폭으로 맞춰집니다.
- 꺼두면 이전 수동 너비로 돌아갑니다.
- 설정이 저장되어 다음 실행 때도 유지됩니다.
- 되돌리기/다시실행 후에도 폭이 망가지지 않습니다.

---

## 17) 문제 색상 표시

표 안의 행들은 단순히 텍스트만 들어 있는 것이 아니라,  
연결된 문제 기준으로 색상 구분이 될 수 있습니다.

즉, 같은 문제와 관련된 항목은 보기 쉽게 묶여 보일 수 있습니다.

사용자는 이걸 통해:
- 같은 문제에 속한 내용인지
- 어떤 줄끼리 연결되는지

좀 더 쉽게 파악할 수 있습니다.

## 18) 백업 기능

작업 내용은 백업할 수 있습니다.

### 백업하기
하단의 **백업하기** 버튼을 누르면  
현재 작성 중인 표를 백업 파일로 저장합니다.

백업은:
- 작업 중 실수 대비
- 이전 버전 보관
- 다른 날 작업 이어서 하기

에 유용합니다.

---

### 백업 목록
하단 목록에서 저장된 백업 파일을 선택할 수 있습니다.

최신 백업 위주로 관리됩니다.

---

### 백업 파일 열기
백업 파일 열기 버튼을 누르면  
선택한 백업 파일을 별도 백업창으로 열 수 있습니다.

이 창은 현재 작업창과 나란히 보면서 비교하기 좋습니다.

---

## 19) 백업창에서 할 수 있는 일

백업창은 **보기 / 복사 중심** 창입니다.

여기서는 보통:

- 예전 백업 확인
- 필요한 답안 복사
- 현재 작업창에 붙여넣기
- 자동복사 기능 사용

을 하게 됩니다.

직접 편집하는 용도가 아니라  
**참조용 / 가져오기용**이라고 생각하면 됩니다.

## 20) 백업창 → 메인창 추가정답 자동복사

이 기능은 실제로 매우 유용합니다.

### 사용 방법
1. 메인 작업창을 열어 둡니다.
2. 백업 파일을 백업창으로 엽니다.
3. 백업창에서 **일치 행 추가정답 자동복사** 버튼을 누릅니다.

---

### 어떤 식으로 복사되나요?
- 백업 파일의 기본정답
- 메인창의 기본정답

이 정확히 같은 줄을 자동으로 찾아서  
**추가정답만 메인창으로 복사**합니다.

---

### 이 기능의 장점
- 예전 백업에 있던 별칭을 다시 손으로 옮길 필요가 없습니다.
- 이미 메인창에 있는 값은 중복으로 넣지 않습니다.
- 필요한 값만 자동으로 합쳐 줍니다.

---

### 열이 부족한 경우
백업 쪽에 추가정답 열이 더 많으면  
메인창의 열도 자동으로 늘려서 복사합니다.

사용자가 먼저 열을 수동으로 늘릴 필요가 없습니다.

---

### 백업에 없는 기본정답
백업에 없는 항목은 자동으로 건너뜁니다.

복사 후 건너뛴 항목이 있으면  
그 위치로 이동해서 사용자가 확인하기 쉽도록 해 줍니다.

---

## 21) 백업창의 복사 옵션

백업창에서는 아래 옵션을 상황에 따라 사용할 수 있습니다.

### 문제 단위로 복사
현재 선택한 문제 전체를 한 번에 복사합니다.

---

### 추가정답만 복사
기본정답을 제외하고 추가정답 칸만 복사합니다.

---

### 예시
예전 백업의 별칭만 현재 표에 가져오고 싶다면  
이 옵션을 켜두면 편합니다.

---

## 22) 작업 내용 임시 저장

메인 작업창을 닫으면 현재 작업 내용이 임시 저장됩니다.  
그래서 실수로 닫아도 다음에 다시 열었을 때 이어서 작업할 수 있습니다.

즉, 백업 버튼을 누르지 않았더라도  
최근 작업 상태가 어느 정도 유지됩니다.

---

## 23) 프로그램이 예전 백업도 최대한 읽어줌

예전에 만들어 둔 백업 파일이 있어도  
프로그램이 가능한 범위에서 자동으로 읽고 맞춰 줍니다.

다만 아주 오래된 형식의 백업은 일부 기능이 제한될 수 있습니다.

그럴 때는:
- 백업 내용을 한 번 열어 확인하고
- 새 백업으로 다시 저장해 두는 것이 좋습니다.

---

## 24) 최종 생성하기

### 생성 방법
1. 표에 정답을 모두 정리합니다.
2. 하단의 **챗 이벤트 생성** 버튼을 누릅니다.
3. 챗 이벤트 파일이 자동으로 생성됩니다.

---

### 빠른 생성 단축키
- `Ctrl + S`

---

### 생성 전에 확인되는 것
생성하기 전에 프로그램은
- 글자 수 제한
- 표 상태

등을 다시 확인합니다.

문제가 있으면 생성하지 않고 알려줍니다.

---

## 25) 추천 작업 순서

처음 사용하는 분이라면 아래 순서로 하면 편합니다.

1. 검색으로 기존 답안이 있는지 확인
2. 같은 의미의 답안은 한 줄에 묶어서 입력
3. 필요한 만큼 추가정답 열을 늘려 입력
4. 중복 사용 중인 답안은 `!` 표시로 확인
5. 필요하면 백업 저장
6. 예전 백업에서 필요한 별칭은 자동복사 기능 활용
7. 마지막으로 생성 버튼 또는 `Ctrl+S` 실행

---

## 26) 자주 헷갈리는 부분 정리

### Q. 같은 답안을 다른 문제에도 써도 되나요?
네. 이제 가능합니다.  
프로그램이 자동으로 중복 정답으로 처리합니다.

---

### Q. `!` 표시는 오류인가요?
아닙니다.  
중복 사용 중임을 알려주는 **확인 표시**입니다.

---

### Q. 기본정답과 추가정답의 차이는 뭔가요?
기본정답은 대표 이름이고,  
추가정답은 별칭 / 약칭 / 다른 표기라고 생각하면 됩니다.

---

### Q. 백업창은 편집용인가요?
아니요.  
주로 확인 / 복사 / 자동복사 용도입니다.

---

### Q. 열너비 자동맞춤을 껐다 켜도 유지되나요?
네.  
설정이 저장되므로 다음 실행 때도 그대로 유지됩니다.

---

## 27) 단축키 요약

- `Ctrl + F` : 검색칸으로 이동
- `Enter` / `F3` : 다음 검색 결과 이동
- `Esc` : 검색어 지우기
- `Ctrl + S` : 챗 이벤트 생성
- `Ctrl + C` : 복사
- `Ctrl + V` : 붙여넣기
- `Ctrl + X` : 잘라내기
- `Delete` : 선택 셀 비우기
- `Ctrl + Z` : 실행 취소
- `Ctrl + Y` : 다시 실행
- `Enter` : 다음 행의 빈칸으로 이동

---

## 28) 한 줄 요약

이 창은  
**채팅 정답과 별칭을 표로 정리하고, 중복 정답도 자동 인식하면서, 백업 기능까지 함께 지원하는 챗 이벤트 전용 관리 도구**입니다.

---

## 29) 마지막으로 꼭 기억하면 좋은 점

- 같은 의미는 한 줄에 묶기
- 중복 정답은 이제 사용 가능
- `!` 표시는 오류가 아니라 확인 표시
- 백업창 자동복사 기능이 매우 유용
- 열너비 자동맞춤으로 보기 편하게 관리 가능
- 생성은 `Ctrl+S`로 빠르게 가능
