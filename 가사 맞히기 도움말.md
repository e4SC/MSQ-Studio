# 🎵 가사 입력 / 관리 시스템 도움말

이 문서는 **노래 가사 추가 및 수정 화면**의 모든 기능을 사용자 관점에서 정리한 전체 가이드입니다.  

---

# 1. 화면 구성 개요

이 화면은 크게 다음 영역으로 구성되어 있습니다.

1. 상단 설정 영역  
2. 노래 기본 정보 영역 (추후에 업데이트 예정)
3. 가사 페이지 영역  
4. 하단 저장 버튼  

---

# 2. 상단 설정 영역

상단에는 다음 기능들이 있습니다.

### ▸ 정보 저장 체크박스
- 체크하면 저장 후에도 현재 입력 내용이 유지됩니다.
- 체크하지 않으면 저장 후 입력창이 초기화됩니다.

### ▸ 길이
- 노래 재생 길이를 입력합니다.
- 숫자만 입력 가능합니다.
- 1 이상의 값만 허용됩니다.

### ▸ 빈칸 일괄생성
- 모든 페이지의 가사 정답을 기준으로 빈칸을 자동 생성합니다.
- 한글, 영어, 숫자는 `_`로 변환됩니다.
- 공백은 그대로 유지됩니다.

### ▸ 페이지 생성
- 새로운 가사 페이지를 추가합니다.
- 생성 즉시 해당 페이지가 열립니다.

### ▸ 페이지 삭제
- 마지막 페이지를 삭제합니다.
- 1페이지는 삭제할 수 없습니다.
- 입력된 값이 있으면 확인 메시지가 표시됩니다.

---

# 3. 노래 기본 정보 영역  
*(현재 비활성화 상태 – 추후 업데이트 예정)*

이 영역은 곡의 기본 정보를 입력하는 공간이지만,  
**현재 버전에서는 사용이 제한되어 있습니다.**

화면에는 표시되지만,  
설정에 따라 자동으로 비활성화되어 입력이 불가능할 수 있습니다.

---

## 📌 예정 기능 안내 (현재 사용 불가)

| 항목 | 설명 |
|------|------|
| 주제 | 곡의 분류 또는 테마 |
| 노래 | 실제 곡 제목 |
| 가수1~7 | 참여 가수 입력 (최대 7명) |
| 힌트 | 각 항목에 대한 보조 설명 |

---

## ⚠️ 현재 상태

- 해당 영역은 내부적으로 준비되어 있으나  
  **사용자가 직접 입력하는 기능은 현재 지원되지 않습니다.**
- 일부 항목은 자동으로 **“(사용안함)”**으로 표시됩니다.
- 입력 규칙 및 세부 기능은 향후 업데이트에서 활성화될 예정입니다.

---

## 🔄 향후 업데이트 예정

다음 업데이트에서 정식 활성화가 계획되어 있습니다:

- 기본 정보 입력 기능 개방
- 가수 입력 구조 개선
- 힌트 입력 방식 개선
- 사용자 편의성 강화
- UI 구성 최적화

※ 본 영역은 현재 버전에서는 참고용으로만 존재하며,  
정식 기능은 추후 업데이트를 통해 제공됩니다.

---

# 4. 가사 페이지 구조

각 페이지는 다음 구조로 구성됩니다.

- 정답
- 빈칸
- 번역

각 영역은 4줄로 구성되어 있습니다.  
한 줄당 최대 20칸까지 입력 가능합니다.

---

## 4-1. 바이트 제한

각 줄은 **110바이트를 초과할 수 없습니다.**

- 영어/숫자/공백 → 1바이트
- 한글/일본어/특수문자 → 3바이트

110바이트를 초과하면:

- 바이트 표시가 빨간색으로 변합니다.
- 해당 줄의 글자가 빨간색으로 표시됩니다.
- 저장이 불가능합니다.

---

## 4-2. 정답 / 빈칸 / 번역 개수 규칙

한 줄에서:

- 정답 개수
- 빈칸 개수
- 번역 개수

이 세 개는 반드시 서로 같아야 합니다.

예시:

| 정답 | 빈칸 | 번역 |
|------|------|------|
| 3개 | 3개 | 3개 |

개수가 맞지 않으면 저장이 불가능합니다.

---

# 5. 가사 입력 편의 기능

## ▸ 붙여넣기 자동 분리

문장을 통째로 붙여넣으면 **공백(스페이스) 기준으로 자동 분리**되어  
각 칸에 순서대로 입력됩니다.

---

### 📌 예시 1

**붙여넣은 문장**

```
나는 오늘도 너를 생각해
```

**자동 분리 결과**

| 1칸 | 2칸 | 3칸 | 4칸 | 5칸 |
|------|------|------|------|------|
| 나는 | 오늘도 | 너를 | 생각해 | (빈칸) |

---

### 📌 예시 2

**붙여넣은 문장**

```
I will always love you
```

**자동 분리 결과**

| 1칸 | 2칸 | 3칸 | 4칸 | 5칸 |
|------|------|------|------|------|
| I | will | always | love | you |

---

### 📌 예시 3 (줄바꿈 포함)

**붙여넣은 문장**

```
이 밤이 지나면
우리는 다시
```

**자동 분리 기준**

- 공백
- 탭
- 줄바꿈

**자동 분리 결과 (한 줄 기준)**

| 1칸 | 2칸 | 3칸 | 4칸 |
|------|------|------|------|
| 이 | 밤이 | 지나면 | 우리는 |

(남은 단어는 다음 줄 또는 이후 칸에 이어서 입력됩니다.)

---

### ⚠️ 참고 사항

- 공백을 기준으로 단어가 나뉩니다.
- 연속 공백은 자동으로 하나의 구분자로 처리됩니다.
- 20칸을 초과하는 경우, 초과 단어는 자동 입력되지 않습니다.


## ▸ Enter 이동
엔터를 누르면 다음 줄 첫 칸으로 이동합니다.

---

# 6. 페이지 우클릭 메뉴

페이지 영역에서 우클릭 시 다음 기능이 있습니다.

| 기능 | 설명 |
|------|------|
| 페이지 복사 | 현재 페이지 내용을 임시 저장 |
| 페이지 붙여넣기 | 복사한 페이지 내용 적용 |
| 페이지 생성 | 새 페이지 추가 |
| 페이지 삭제 | 현재 페이지 삭제 |
| 빈칸 일괄생성 | 해당 페이지의 빈칸 자동 생성 |
| 페이지 비우기 | 현재 페이지 모든 내용 삭제 |

---

# 7. 자동 변환 기능

## ▸ 빈칸 생성
정답을 기준으로 `_` 형태로 자동 변환됩니다.

예:
```
안녕 하세요
→ _ _  _ _ _
```

## ▸ 초성 변환 (추후 업데이트 예정)
정답을 기준으로 초성만 추출합니다.

예:
```
사랑해
→ ㅅㄹㅎ
```

영어 입력도 한글 자판 초성 기준으로 변환됩니다.

---

# 8. 자동완성 기능

입력 중 기존 데이터와 일치하는 항목이 있으면 자동 추천 목록이 표시됩니다.

### 사용 방법

- 글자를 입력하면 목록이 나타납니다.
- 방향키로 선택
- Enter 또는 Tab으로 확정

---

# 9. 저장 동작

저장 버튼을 누르면 다음 항목을 검사합니다.

1. 110바이트 초과 여부
2. 정답/빈칸/번역 개수 일치 여부
3. 길이 값 유효성
4. 필수 항목 입력 여부
5. 설정에 따른 사용 가능 여부

문제가 없으면 파일에 저장됩니다.

---

# 10. 수정 모드

수정 모드에서는:

- 기존 데이터를 불러옵니다.
- 저장 시 기존 내용을 교체합니다.
- 변경 사항이 있으면 종료 시 저장 여부를 묻습니다.

---

# 11. 종료 동작

### ▸ 추가 모드
입력 내용이 있으면 종료 전 확인 메시지가 표시됩니다.

### ▸ 수정 모드
변경 내용이 있으면:
- 저장 후 종료
- 저장 없이 종료
- 취소

중 선택할 수 있습니다.

---

# 12. 자동 복구 및 동기화

저장 시:

- 가사 설정 파일이 자동 갱신됩니다.
- 관련 텍스트 파일이 동기화됩니다.
- 백업 폴더가 자동 생성될 수 있습니다.

---

# 13. 입력 제한 요약

| 항목 | 제한 |
|------|------|
| 가사 한 줄 | 110바이트 |
| 가사 한 줄 최대 칸 | 20칸 |
| 가사 한 페이지 | 4줄 |

---

# 14. 단축키

| 단축키 | 기능 |
|--------|------|
| Ctrl + S | 저장 |
| Enter | 다음 줄 이동 |

---

# 15. 주의 사항

- 빈칸/번역은 정답 개수와 반드시 일치해야 합니다.
- 1페이지는 삭제할 수 없습니다.
- 길이는 1 이상이어야 합니다.
- 바이트 초과 시 저장 불가합니다.

---

# 16. 전체 흐름 요약

1. 길이 입력
2. 노래 기본 정보 입력 (추후 업데이트 예정)
3. 가사 정답 입력
4. 빈칸/번역 생성
5. 바이트 확인
6. 저장
