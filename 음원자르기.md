# ✂️ 음원 자르기 사용 설명서
## 처음 사용하는 분도 그대로 따라하면 되는 전체 안내서

이 문서는 **음원 자르기 기능을 처음 사용하는 사용자 기준**으로 작성된 설명서입니다.  
프로그램 화면에서 보이는 기능을 중심으로, **무엇을 어디에 입력해야 하는지**, **어떤 순서로 써야 하는지**, **자주 헷갈리는 부분이 무엇인지**까지 한 번에 이해할 수 있도록 정리했습니다.

---

# 1. 이 기능으로 할 수 있는 것

음원 자르기 기능에서는 아래 작업을 할 수 있습니다.

- 유튜브 링크를 넣고 원하는 구간만 잘라서 저장
- 내 컴퓨터에 있는 음원 파일을 직접 불러와서 저장
- 저장 형식을 MP3 / OGG / WAV 중에서 선택
- 여러 곡을 한 번에 자르기
- 자른 음원이 실제로 잘 들어갔는지 검사
- 누락되거나 잘못된 곡만 다시 자르기
- 최근 자른 곡 확인
- 불필요하게 남은 파일 정리

그리고 가장 중요한 점은 다음과 같습니다.

> **이제 음원은 자르기만 하면 자동으로 삽입용 구조까지 같이 처리됩니다.**  
> 즉, 예전처럼 음원을 따로 넣기 위해 다른 작업을 다시 할 필요가 없습니다.

---

# 2. 시작 전에 꼭 알아두세요

## 2-1. 행 번호의 의미

음원 정보 표의 번호는 아래처럼 사용됩니다.

- **0번 행 = 오프닝**
- **1번 행 = 엔딩**
- **2번 행부터 = 실제 문제곡**

즉:

- 2번 행 = 1번 노래
- 3번 행 = 2번 노래
- 4번 행 = 3번 노래

이 구조로 생각하시면 됩니다.

---

## 2-2. 오프닝 / 엔딩은 반드시 작성해야 합니다

이 프로그램은 **0번 행과 1번 행을 일반 곡이 아니라 오프닝 / 엔딩으로 사용**합니다.

따라서 처음 작업할 때는 반드시:

- **0번 행에 오프닝 곡**
- **1번 행에 엔딩 곡**

을 입력하고 자르셔야 합니다.

---

## 2-3. 음원은 “불러오기만” 하면 끝이 아닙니다

유튜브 링크를 넣든, 로컬 파일을 불러오든,  
**표에 값만 들어간 상태는 아직 맵에 반영된 상태가 아닙니다.**

반드시 마지막에 **자르기 버튼을 눌러 실제로 자르기 작업까지 해야** 반영됩니다.

---

## 2-4. 지원되는 음원 입력 방식

이 기능은 두 가지 방식으로 음원을 넣을 수 있습니다.

### 1) 유튜브 링크
- 일반 유튜브 링크
- 짧은 링크
- 재생목록 링크

### 2) 로컬 파일
- `.mp3`
- `.ogg`
- `.wav`

즉, 이미 가지고 있는 음원 파일도 직접 불러와서 사용할 수 있습니다.

---

# 3. 화면 구성 설명

음원 자르기 창은 크게 **왼쪽 설정 영역**과 **오른쪽 음원 정보 입력 표**로 나뉩니다.

---

## 3-1. 왼쪽 상단: 출력 설정

여기서는 저장 형식을 정합니다.

- **MP3 (추천)**
- **OGG**
- **WAV**

한 번에 하나만 선택됩니다.

### MP3 선택 시
아래 설정을 추가로 조절할 수 있습니다.

- 샘플레이트
- 비트레이트
- 채널(모노 / 스테레오)

### OGG / WAV 선택 시
이 세부 설정은 직접 바꾸지 않는 방식으로 동작합니다.

---

## 3-2. 왼쪽 중단: 자르기 설정

여기에는 다음 값을 입력합니다.

- **시작 번호**
- **끝 번호**
- **볼륨**
- **동시 자르기 개수**

### 시작 번호 / 끝 번호
어느 행부터 어느 행까지 자를지 정하는 값입니다.

### 볼륨
잘라낸 음원에 적용할 음량입니다.

### 동시 자르기 개수
한 번에 몇 개까지 동시에 처리할지 정하는 값입니다.

- 입력 범위: **1 ~ 5**
- 3 이상으로 설정하면 성능 경고가 뜰 수 있습니다.

권장:
- 보통은 **1~2개**가 가장 안정적입니다.
- 3~5개는 컴퓨터 사양에 따라 CPU 사용량이 많이 올라갈 수 있습니다.

---

## 3-3. 왼쪽 버튼 영역

왼쪽에는 주요 버튼들이 있습니다.

- **자르기**
- **음원 정보 검사**
- **(현재 형식) 파일 열기**
- **FAQ**
- **도움말**
- **불필요 파일 정리**

---

## 3-4. 왼쪽 하단: 음원 관리 영역

여기서는 현재 선택한 형식 기준으로 아래 정보를 볼 수 있습니다.

- 현재 형식의 파일 개수
- 총 용량
- 불필요 파일 정리 버튼
- 최근 자르기 기록

최근 자르기 기록에서는 최근에 성공적으로 자른 곡들이 표시되며,  
더블클릭하면 해당 번호로 이동해서 다시 확인할 수 있습니다.

---

## 3-5. 오른쪽: 음원 정보 입력 표

오른쪽 표에서 실제 곡 정보를 입력합니다.

표의 열은 다음과 같습니다.

- **번호**
- **노래 이름**
- **시작 시간**
- **종료 시간**
- **YouTube 링크 / 로컬파일**
- **불러오기**

---

# 4. 처음 사용할 때 가장 추천하는 순서

처음 사용하는 분은 아래 순서대로 하시면 가장 쉽게 사용할 수 있습니다.

## 1단계. 오프닝 / 엔딩 입력
- 0번 행에 오프닝 입력
- 1번 행에 엔딩 입력

## 2단계. 2번 행부터 문제곡 입력
- 2번 행부터 본게임에 사용할 노래를 넣기

## 3단계. 링크 또는 로컬 파일 넣기
- 유튜브 링크를 붙여넣거나
- 불러오기 버튼 또는 드래그 앤 드롭으로 로컬 파일 넣기

## 4단계. 제목 / 시작 / 종료 확인
- 제목이 맞는지
- 시작 시간과 종료 시간이 맞는지
- 링크가 올바른지 확인

## 5단계. 자르기 실행
- 왼쪽에서 시작 번호, 끝 번호, 볼륨을 정한 뒤 자르기

## 6단계. 검사
- 필요하면 음원 정보 검사로 누락 / 중복 / 불일치 확인

---

# 5. 음원 넣는 방법

---

## 5-1. 유튜브 링크로 넣기

표의 **YouTube 링크 / 로컬파일** 칸에 유튜브 링크를 붙여넣으면 됩니다.

지원되는 형태:
- 일반 영상 링크
- 짧은 링크
- 일부 재생목록 링크

정상적인 영상 링크는 자동으로 정리된 형태로 바뀔 수 있습니다.

---

## 5-2. 로컬 파일로 넣기

로컬 파일은 두 가지 방식으로 넣을 수 있습니다.

### 방법 A. 불러오기 버튼
오른쪽 표의 **불러오기** 버튼을 눌러 직접 파일을 선택합니다.

### 방법 B. 드래그 앤 드롭
파일을 표에 직접 끌어다 놓습니다.

지원 파일 형식:
- MP3
- OGG
- WAV

로컬 파일을 넣으면:
- 링크 칸에는 파일 경로가 들어가고
- 제목은 파일명 기준으로 자동 정리되어 들어갑니다
- 시작 시간 / 종료 시간은 기본값으로 들어갑니다

---

## 5-3. 여러 로컬 파일 한 번에 넣기

여러 파일을 한 번에 불러오거나 드래그 앤 드롭하면  
**시작 번호를 지정해서 순서대로 넣을 수 있습니다.**

예를 들어:

- 10개의 파일 선택
- 시작 번호를 2로 지정

그러면 2번 행부터 아래로 차례대로 들어갑니다.

주의:
- 이미 값이 들어 있는 행은 건너뛰고
- 다음 빈 행부터 계속 들어갑니다

즉, 기존 입력을 덮어쓰지 않도록 배려된 방식입니다.

---

# 6. 번호 구조를 정확히 이해하기

이 부분을 가장 많이 헷갈려 하니 꼭 확인하세요.

| 행 번호 | 의미 |
|---|---|
| 0 | 오프닝 |
| 1 | 엔딩 |
| 2 | 1번 문제곡 |
| 3 | 2번 문제곡 |
| 4 | 3번 문제곡 |

따라서:

- **1번 문제곡만 자르려면 2번 행을 자르는 것**
- **1번~10번 문제곡을 자르려면 2번 행부터 11번 행까지 자르는 것**

입니다.

---

# 7. 시간 입력 방법

시작 시간 / 종료 시간은 기본적으로 **mm:ss 형식**으로 입력합니다.

예:
- `0:15`
- `1:20`
- `03:45`

---

## 7-1. 자동 보정되는 경우

일부 입력은 자동으로 시간 형식으로 바뀔 수 있습니다.

예:
- `123` → `1:23`
- `0320` → `3:20`

또한 일부 기호 입력도 자동 보정될 수 있습니다.

---

## 7-2. 시간 입력 시 주의사항

다음 조건을 만족해야 합니다.

- 분 / 초는 각각 정상 범위여야 함
- 시작 시간은 종료 시간보다 반드시 작아야 함

잘못 입력하면:
- 경고가 뜨고
- 해당 칸 값이 지워지거나 다시 입력을 요구할 수 있습니다

---

# 8. 제목 입력 시 주의사항

제목에는 일부 특수문자가 자동으로 제거될 수 있습니다.

즉, 제목에 사용하기 부적절한 문자가 들어 있으면:
- 자동으로 정리되고
- 필요하면 안내 메시지가 뜰 수 있습니다

로컬 파일을 불러온 경우에도 파일명을 기준으로 제목이 자동 보정될 수 있습니다.

---

# 9. 출력 형식 설명

---

## 9-1. MP3
가장 일반적으로 추천되는 형식입니다.

장점:
- 관리가 편함
- 설정 변경 가능
- 보통 사용하기 무난함

MP3를 선택하면 다음을 조절할 수 있습니다.

- 샘플레이트
- 비트레이트
- 모노 / 스테레오

---

## 9-2. OGG
OGG 형식으로 저장합니다.

장점:
- 용량 효율이 괜찮음

직접 세부 설정을 바꾸는 방식보다는,  
프로그램이 적절한 구조로 처리하는 쪽에 가깝습니다.

---

## 9-3. WAV
WAV 형식으로 저장합니다.

특징:
- 비교적 원형에 가까운 저장 방식
- 세부 설정은 직접 조정하는 방식이 아님

---

# 10. 볼륨 설정

볼륨 칸에는 숫자만 입력할 수 있습니다.

## 10-1. 입력 가능한 값
- 숫자만 입력 가능
- 너무 낮거나 높으면 경고가 뜰 수 있음
- 20을 초과하면 진행되지 않습니다

## 10-2. 권장 범위
프로그램에서도 안내하듯이 대체로 아래 범위를 기준으로 생각하시면 됩니다.

- **14 ~ 18 권장**

이 범위를 벗어나면:
- 계속 진행할지 한 번 더 묻는 경고창이 뜰 수 있습니다

---

# 11. 자르기 실행 방법

---

## 11-1. 범위 자르기

왼쪽의:

- 시작 번호
- 끝 번호

를 직접 입력해서 자르는 방식입니다.

예:
- 시작 번호: 2
- 끝 번호: 11

이면 2번 행부터 11번 행까지 자릅니다.

---

## 11-2. 선택 자르기

오른쪽 표에서 여러 행을 선택하면  
버튼이 상황에 따라 **선택 값 자르기**처럼 동작할 수 있습니다.

즉:
- 전체 범위를 입력하지 않아도
- 선택한 행만 기준으로 자르기 가능

우클릭 메뉴의 **자르기**를 통해서도 사용할 수 있습니다.

---

## 11-3. 자르기 전에 확인되는 것

자르기를 누르면 프로그램이 먼저 아래를 검사합니다.

- 시작 번호 / 끝 번호 / 볼륨이 숫자인지
- 번호가 음수가 아닌지
- 끝 번호가 시작 번호보다 작은지
- 시작 위치와 끝 위치의 제목이 비어 있지 않은지
- 볼륨이 허용 범위 안인지

문제가 있으면 그 자리에서 멈추고 안내 메시지가 뜹니다.

---

## 11-4. 자르기 확인 창

실행 전에 확인 창이 뜹니다.

여기서는 다음을 확인할 수 있습니다.

- 현재 저장 형식
- 몇 개를 자르는지
- 볼륨
- 자를 대상 목록 전체

즉, 어떤 곡이 실제로 자를 대상에 들어가는지 스크롤해서 확인한 뒤 진행할 수 있습니다.

---

# 12. 자르기 진행 창

자르기를 시작하면 **진행 창**이 따로 열립니다.

이 창에서는 다음을 볼 수 있습니다.

- 현재 전체 진행 개수
- 각 곡별 상태
- 각 곡별 진행률
- 취소 버튼

---

## 12-1. 진행 중 표시 예시

곡마다 상태가 대략 아래처럼 바뀔 수 있습니다.

- 준비 중
- 길이 계산 중
- 자르는 중
- 변환 중
- 정규화 중
- 분할 중
- 완료
- 실패

---

## 12-2. 취소 방법

진행 창 아래의 **취소** 버튼을 누르면  
즉시 중단되는 것이 아니라:

> **현재 처리 중인 작업이 끝난 뒤 중지**

됩니다.

즉, 강제 종료형이 아니라 **안전하게 중단 요청**하는 방식입니다.

작업 중에는 창을 마음대로 닫을 수 없고,  
닫으려고 하면 안내 메시지가 뜹니다.

---

# 13. 작업 결과 창

자르기가 끝나면 결과 요약 창이 뜹니다.

여기서는:

- 성공한 곡 목록
- 실패한 곡 목록
- 실패 사유
- 중간에 취소되었는지 여부

를 확인할 수 있습니다.

또한 결과를 복사할 수도 있습니다.

---

# 14. 음원 정보 검사 기능

왼쪽의 **음원 정보 검사** 버튼으로 검사할 수 있습니다.

이 기능은 현재 선택한 형식 폴더를 기준으로,  
입력된 표의 정보와 실제 저장된 파일을 비교합니다.

---

## 14-1. 검사 항목

검사에서는 대표적으로 아래 문제를 찾아냅니다.

### 1) 파일 누락
표에는 정보가 있는데 실제 파일이 없음

### 2) 같은 번호의 중복 파일
같은 번호로 잘린 파일이 2개 이상 존재

### 3) 제목 불일치
표의 제목과 실제 파일 제목이 다름

### 4) 파일만 있고 표 정보가 없음
파일은 있는데 표 쪽 제목이 비어 있음

---

## 14-2. 검사 결과 창에서 할 수 있는 것

검사 결과 창에서는 다음 기능을 사용할 수 있습니다.

- 전체 결과 보기
- 누락된 곡 목록만 보기
- 전체 복사
- 누락 재자르기
- 선택 재자르기

즉, 문제가 있는 곡만 골라서 다시 자를 수 있습니다.

---

# 15. 최근 자르기 기록

왼쪽 하단의 **최근 자르기 기록**에는  
최근에 성공적으로 잘린 곡 목록이 표시됩니다.

특징:
- 최근 성공한 곡만 표시
- 번호와 제목 확인 가능
- 더블클릭하면 해당 곡 위치로 이동 가능

즉, 방금 작업한 곡을 다시 찾기 쉽습니다.

---

# 16. 불필요 파일 정리

음원을 여러 번 자르다 보면 아래 같은 파일이 남을 수 있습니다.

- 중복 파일
- 제목이 안 맞는 파일
- 표 정보 없는 파일
- 규칙에 맞지 않는 파일
- 크기가 비정상인 파일

이럴 때 **불필요 파일 정리** 기능을 사용할 수 있습니다.

---

## 16-1. 정리 창에서 할 수 있는 것

정리 창에서는:

- 삭제 대상 보기
- 체크해서 선택 삭제
- 전체 선택 / 해제
- 전체 삭제
- 필터로 보기
- 선택 파일 미리보기 / 열기

를 할 수 있습니다.

---

## 16-2. 필터 종류

정리 창에서는 상황에 따라 아래처럼 걸러서 볼 수 있습니다.

- 전체 보기
- 중복 파일
- 제목 불일치
- 음원정보 없음

---

# 17. 오른쪽 표 사용법

이 표는 엑셀처럼 다룰 수 있습니다.

---

## 17-1. 기본 편집

다음 값을 직접 입력할 수 있습니다.

- 제목
- 시작 시간
- 종료 시간
- 링크 / 로컬 파일

번호 열은 자동으로 관리되므로 직접 수정하는 용도가 아닙니다.

---

## 17-2. 붙여넣기 지원

표에서는 여러 셀을 한 번에 붙여넣을 수 있습니다.

지원 동작:
- 복사
- 잘라내기
- 붙여넣기
- Delete / Backspace로 값 비우기
- Undo / Redo

즉, 외부에서 정리한 데이터를 한 번에 옮기기 편합니다.

---

## 17-3. 전체 선택 관련

표에서는 링크가 들어간 행들만 빠르게 선택하는 동작도 지원됩니다.  
작업 범위가 많은 경우 한꺼번에 관리하기 편합니다.

---

# 18. 표 검색 기능

오른쪽 아래 검색 칸에서 제목을 검색할 수 있습니다.

동작 방식:
- 검색어 입력 시 일치하는 제목이 강조 표시됨
- Enter로 다음 결과로 이동
- Esc로 검색 강조 해제

즉, 곡 수가 많을 때 특정 곡을 빠르게 찾는 용도로 사용하면 됩니다.

---

# 19. 링크 중복 표시 기능

표 아래의 **링크 중복 표시**를 켜면  
같은 유튜브 링크를 여러 줄에 넣은 경우 색으로 강조됩니다.

이 기능은:
- 중복을 자동 삭제하는 기능은 아니고
- **중복 여부를 눈에 띄게 보여주는 기능**입니다

실수로 같은 영상을 여러 줄에 넣었는지 확인할 때 유용합니다.

---

# 20. 우클릭 메뉴 기능

표에서 우클릭하면 여러 기능을 사용할 수 있습니다.

---

## 20-1. Undo / Redo
- 이전 작업 되돌리기
- 다시 적용하기

## 20-2. 링크 중복 표시
- 중복 강조 켜기 / 끄기

## 20-3. 행 삭제
- 선택한 행 삭제
- 단, 오프닝 / 엔딩 행은 삭제 불가

## 20-4. 행 추가
- 현재 위치 다음에 빈 행 추가

## 20-5. 링크 열기
- 브라우저에서 링크 열기

## 20-6. 링크 열기(시작시간)
- 입력된 시작 시간 기준으로 링크 열기

## 20-7. 자르기
- 선택한 행만 바로 자르기

## 20-8. 선택된 값만 삭제
- 선택된 칸만 비우기

## 20-9. 전체 삭제
- 표 안의 입력값 전체 비우기

---

# 21. 단축키

표 편집과 검색에 자주 쓰는 단축키는 아래와 같습니다.

- `Ctrl + C` : 복사
- `Ctrl + X` : 잘라내기
- `Ctrl + V` : 붙여넣기
- `Ctrl + Z` : 실행 취소
- `Ctrl + Y` : 다시 실행
- `Ctrl + F` : 검색 칸으로 이동
- `Delete / Backspace` : 선택 셀 비우기
- `Esc` : 선택 해제 / 검색 해제 / 일부 창 닫기

추가로 메인 창에서는 다음도 사용할 수 있습니다.

- `Esc` : 상황에 따라 편집 취소 / 작업 취소 요청 / 창 닫기

---

# 22. 파일 저장 위치

선택한 출력 형식에 따라 저장 위치가 달라집니다.

- MP3 → `sound/mp3`
- OGG → `sound/ogg`
- WAV → `sound/wav`

버튼으로 바로 열 수 있습니다.

---

# 23. 파일 이름 규칙

저장되는 파일은 기본적으로 아래와 같은 형식을 따릅니다.

- `번호. 제목.확장자`

예:
- `2. 노래제목.mp3`

같은 번호 / 제목 파일이 이미 있으면:
- `2_1. 노래제목.mp3`
- `2_2. 노래제목.mp3`

처럼 자동으로 중복을 피해 저장됩니다.

---

# 24. 로컬 파일 사용할 때 주의사항

로컬 파일도 사용할 수 있지만, 아래 점을 기억해 주세요.

- 로컬 파일을 넣는 것만으로 끝나지 않음
- 반드시 마지막에 자르기 실행 필요
- 제목은 파일명 기준으로 자동 정리될 수 있음
- 기존 원본 파일 자체를 수정하는 방식이 아니라, 별도 결과 파일을 생성하는 방식

즉, 원본 파일을 망가뜨리지 않고 결과물을 새로 만드는 구조입니다.

---

# 25. 처음 사용하는 분께 추천하는 실제 예시

예를 들어 곡 3개와 오프닝 / 엔딩을 넣는다고 하면:

## 1단계
0번 행에 오프닝 입력

## 2단계
1번 행에 엔딩 입력

## 3단계
2번 행, 3번 행, 4번 행에 문제곡 입력

## 4단계
각 행에:
- 제목
- 시작 시간
- 종료 시간
- 링크 또는 로컬 파일

을 넣기

## 5단계
왼쪽에서:
- 시작 번호: 0
- 끝 번호: 4
- 볼륨: 16 정도
- 형식: MP3

설정

## 6단계
자르기 클릭

## 7단계
확인 창에서 대상 목록 확인 후 예 클릭

## 8단계
완료 후 결과 창 확인

## 9단계
필요하면 음원 정보 검사 실행

# 26. 자주 헷갈리는 질문

## Q. 1번 문제곡은 몇 번 행인가요?
**2번 행입니다.**

## Q. 오프닝 / 엔딩은 꼭 해야 하나요?
**네. 0번과 1번은 반드시 작성하는 것을 기준으로 사용해야 합니다.**

## Q. 로컬 파일만 넣으면 끝인가요?
아니요.  
**불러온 후 반드시 자르기를 해야 실제 반영됩니다.**

## Q. 곡 여러 개를 한 번에 넣을 수 있나요?
네.  
드래그 앤 드롭 또는 다중 불러오기로 가능합니다.

## Q. 중간에 값이 있는 행이 있으면 어떻게 되나요?
여러 파일 삽입 시 **그 행은 건너뛰고 다음 빈 행부터 들어갑니다.**

## Q. 잘못 자른 곡만 다시 자를 수 있나요?
네.  
검사 결과 창이나 선택 자르기를 통해 가능합니다.

## Q. 작업 중 창을 그냥 닫아도 되나요?
작업 중에는 닫을 수 없고, **취소 요청 후 안전하게 중단**하는 방식입니다.

# 27. 마지막 요약

이 기능은 아래처럼 이해하시면 됩니다.

- **왼쪽**: 저장 형식 / 자르기 설정 / 검사 / 관리
- **오른쪽**: 곡 정보 입력 표
- **0번 행 = 오프닝**
- **1번 행 = 엔딩**
- **2번 행부터 실제 문제곡**
- 링크든 로컬 파일이든 **넣고 끝이 아니라 반드시 자르기까지 해야 함**
- 자르기 후에는 결과 확인, 필요하면 검사까지 진행하면 됨
